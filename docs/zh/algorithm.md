# 算法说明

`openvad` 是一个基于帧的自适应 VAD。目标是在批处理和低延迟场景下提供
快速、轻量、行为可解释的语音活动检测基线。

## 流程

1. 将输入音频转换为单声道 `float32`，取值通常在 `[-1, 1]`。
2. 按重叠窗口切帧，默认 20 ms 帧长、10 ms 帧移。
3. 计算每帧 RMS 能量、dBFS 能量、过零率和峰均比。
4. 通过较低能量分位数估计初始噪声底。
5. 根据相对噪声底的能量增量换算语音概率。
6. 对脉冲噪声、近直流信号、高过零噪声进行惩罚。
7. 使用进入/退出两个阈值做迟滞判决。
8. 填补短静音、删除短语音岛，并对最终语音段做边界 padding。

## 设计原因

纯能量 VAD 很快，但在边界处容易抖动，也难以处理背景噪声漂移。`openvad`
保留能量法的速度，同时增加三个稳定机制：

- 自适应噪声跟踪主要在非置信语音区间更新。
- 迟滞判决使用独立的进入阈值和退出阈值。
- 段落后处理约束最短语音和最短静音时长。

## 复杂度

运行时间与样本数线性相关。内存占用与帧数线性相关，而不是与样本数直接
绑定。C++ 核心负责样本级循环，并向 Python 返回紧凑的 NumPy 数组。

## 适用边界

该检测器不能替代困难声学场景下的神经网络 VAD。以下情况通常会更难：

- 音乐下的人声。
- 快速变化的工业噪声。
- 低信噪比远场麦克风。
- 能量包络接近语音的非语音声音。

建议使用 `VadResult.frames` 中的帧级数组检查失败案例，并针对目标场景调参。
